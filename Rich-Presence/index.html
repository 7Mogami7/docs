    Rich Presence

    Introduction

    Rich Presence is a feature in Xbox Live/PSN that gives you a very, very brief overview of what the player is currently doing in their game. Normally speaking it's only visible to friends of this player as well, and is a short summary containing the level or mode the player is currently on.

    We have a similar feature to RetroAchievements, but there are a number of differences. The emulator will report back to the website every 600 seconds. Similarly, every 600 seconds or so, the 'active players' box on the frontpage will refresh, detailing the last known activity of all online players. If there isn't a rich presence script given, the text will be 'earning achievements' or 'developing achievements' if the memory dialog is open and visible.

    Please note - the data will be about 10-minutes out of date. Rich Presence isn't designed to give you an up-to-the-second reflection of the players status. It really is just a simple overview, which depending on the game would be a level or map name, whether you are ingame or not, what game type you're playing etc. Generally if the information is likely to change rapidly, more than once a minute (i.e. health) it's generally not a good candidate for Rich Presence data. However there are exceptions: things like 'Score' which give you a rough guide to how long the player has been playing are an exception.

    Every time a game is launched, it fetches the achievements in a 'patch' file for the ROM which details all the achievements and memory addresses (and leaderboards) that can be watched for. It will now also request a Rich Presence script for the currently loaded ROM.

    The Rich Presence scripts for each game can be found under the development section on each game's page:

@LevelName(0xfe10), @Score(0xfe28*10_0xfe29*1000_0xfe26*100000) points, @Lives(0xhfe13) lives

    It breaks down into a series of Lookup objects, Format objects and one Display object.

    Lookups

    Lookups are defined like this: 

    Lookup:NameOfLookup
Value1=Text When This Value
Value2=Text When Another Value
...

    We give the Lookup a value, consisting of a series of memory addresses and modifiers. More about this later.

    Format

    Format tables are defined like this:

    Format:Score
FormatType=VALUE

    Begin with Format:, then the name of the format type. On the next line, give FormatType=, then one of the following: VALUE, SCORE (or POINTS), TIME (or FRAMES), SECS, MILLISECS, or OTHER.

    • VALUE: generic value, no leading zeroes.
    • SCORE/POINTS: "000130 points"
    • TIME/FRAMES: value describes the number of frames elapsed, and will be turned into 00:00.00
    • SECS: value describes the number of seconds elapsed, and will be turned into 00:00
    • MILLISECS: value describes the number of millisecs elapsed, and will be turned into 00:00.00

    Display

    Finally, display will be a string that gets shown in the 'Active Players' box on the front page. It refers to the previously defined Lookup and Format objects using a single '@'. It then specifies a name for the lookup or format (case sensitive!), and immediately after, in brackets, a series of memory values specifying what to send to that lookup or format object. I.e.

    @LevelName(0xfe10)

    This means use the Lookup or Format that's called LevelName, and give it whatever value is in 0xfe10.

    @Score(0xfe28*10_0xfe29*1000_0xfe26*100000) points

    This means use the Lookup or Format Score, and give it the sum of: - 0xfe28 times 10, ADD - 0xfe29 times 1000, ADD - 0xfe26 times 100000

    This is similar formatting to the leaderboards formatting.

    Limits

    • 2500 character limit for script
    • 100 character limit for what is displayed
    • Unicode characters are allowed
